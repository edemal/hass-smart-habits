    Replace the stub JS file from Plan 01 with a full vanilla HTMLElement web component. The panel is a single-file component with no external dependencies.

    **Component structure (SmartHabitsPanel extends HTMLElement):**

    **State management:**
    - `this._hass` — HA connection object, set via `set hass(hass)` property
    - `this._data` — response from get_patterns: `{ patterns: [], accepted_patterns: [], stale_automations: [] }`
    - `this._loading` — boolean, true during initial data load
    - `this._error` — error message string or null
    - `this._customizing` — pattern object currently being customized, or null
    - `this._customizeData` — preview response from preview_automation for the customize form
    - `this._customizeHour` — current trigger_hour value in customize form

    **Lifecycle:**
    - `constructor()`: attachShadow, initialize state properties to null/false
    - `set hass(hass)`: Store hass reference. On first call (this._hass was null), call `_loadData()`. Always store latest hass.
    - `connectedCallback()`: Call `_render()` (renders loading state before hass arrives)
    - Do NOT trigger data load from connectedCallback — hass may not be set yet

    **Data loading:**
    - `async _loadData()`: Set _loading=true, render. Call `this._hass.callWS({ type: "smart_habits/get_patterns" })`. On success, store in _data, clear _error. On error, set _error. Set _loading=false, render.

    **Actions (all optimistic — PANEL-05):**
    - `async _acceptPattern(pattern, overrides={})`: Remove pattern from `this._data.patterns` immediately, render. Call accept_pattern WS with pattern fields + overrides. On success, if response has `yaml_for_manual_copy`, show it in a toast/alert. On error, call `_loadData()` to restore.
    - `async _dismissPattern(pattern)`: Remove pattern from `this._data.patterns` immediately, render. Call dismiss_pattern WS. On error, call `_loadData()` to restore.
    - `async _openCustomize(pattern)`: Set `this._customizing = pattern`. Call preview_automation WS. Store result in `this._customizeData`, set `this._customizeHour = pattern.peak_hour`. Render.
    - `_closeCustomize()`: Set `this._customizing = null`, `this._customizeData = null`. Render.
    - `_acceptCustomized()`: Call `_acceptPattern(this._customizing, { trigger_hour: this._customizeHour })`. Close customize.

    **Rendering (_render method using innerHTML):**

    After setting innerHTML, call `_attachEventListeners()` to wire up button click handlers via `this.shadowRoot.querySelectorAll('[data-action]')`.

    **Event listener wiring (_attachEventListeners):**
    After innerHTML assignment, query all `[data-action]` elements and attach click handlers:
    - `accept`: get pattern from `_allPatterns[index]`, call `_acceptPattern(pattern)`
    - `dismiss`: get pattern from `_allPatterns[index]`, call `_dismissPattern(pattern)`
    - `customize`: get pattern from `_allPatterns[index]`, call `_openCustomize(pattern)`
    - `accept-custom`: call `_acceptCustomized()`
    - `cancel-custom`: call `_closeCustomize()`
    - `hour-change`: update `_customizeHour` from input value

    Store a flat reference `this._allPatterns` (all patterns in render order) so data-index maps to the correct pattern object.

    **Helper methods:**
    - `_groupPatterns(patterns)`: Group by pattern_type using mapping: `{ daily_routine: "Daily Routines", temporal_sequence: "Device Chains", presence_arrival: "Arrival Sequences" }`. Sort within each group by confidence descending. Return ordered entries array: `[[label, patterns], ...]` with defined category order (Daily Routines first, Device Chains second, Arrival Sequences third).
    - `_formatHour(h)`: Return `"HH:00"` format (e.g., 7 -> "07:00", 15 -> "15:00")
    - `_formatDate(iso)`: Return human-readable date from ISO string (e.g., "Jan 15, 2026")
