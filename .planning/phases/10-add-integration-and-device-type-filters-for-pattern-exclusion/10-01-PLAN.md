2. Add a module-level helper function `_get_available_integrations(hass)` that:
   - Gets the entity registry via `er.async_get(hass)`
   - Iterates over `hass.states.async_entity_ids()`
   - Filters to entities whose domain (entity_id.split(".")[0]) is in `DEFAULT_ENTITY_DOMAINS`
   - For each, does `registry.async_get(entity_id)` and collects `entry.platform` where entry is not None
   - Returns `sorted(set of integration names)` as a `list[str]`

3. In `async_step_init`, when `user_input is not None`:
   - Add the two new fields to the `data={}` dict passed to `self.async_create_entry`. These are already `list[str]` from the multi-select, so store them directly WITHOUT int() cast:
     ```python
     CONF_EXCLUDED_INTEGRATIONS: user_input.get(CONF_EXCLUDED_INTEGRATIONS, []),
     CONF_EXCLUDED_DOMAINS: user_input.get(CONF_EXCLUDED_DOMAINS, []),
     ```

4. Before building the schema, read current values:
   ```python
   current_excluded_integrations = self.config_entry.options.get(
       CONF_EXCLUDED_INTEGRATIONS, DEFAULT_EXCLUDED_INTEGRATIONS
   )
   current_excluded_domains = self.config_entry.options.get(
       CONF_EXCLUDED_DOMAINS, DEFAULT_EXCLUDED_DOMAINS
   )
   ```

5. Extend the `vol.Schema({...})` dict with two new fields AFTER the existing `CONF_SEQUENCE_WINDOW` field:
   ```python
   vol.Optional(
       CONF_EXCLUDED_INTEGRATIONS,
       default=current_excluded_integrations,
   ): SelectSelector(
       SelectSelectorConfig(
           options=_get_available_integrations(self.hass),
           multiple=True,
           mode=SelectSelectorMode.LIST,
       )
   ),
   vol.Optional(
       CONF_EXCLUDED_DOMAINS,
       default=current_excluded_domains,
   ): SelectSelector(
       SelectSelectorConfig(
           options=list(DEFAULT_ENTITY_DOMAINS),
           multiple=True,
           mode=SelectSelectorMode.LIST,
       )
   ),
   ```

   Use `vol.Optional` (NOT `vol.Required`) so existing config entries without these fields upgrade gracefully.
